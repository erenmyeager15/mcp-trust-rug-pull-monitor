# Seven-day launch monitor

Check once daily for seven days after the three public tasks are published.

Record:

- Actor detail views;
- input-page views;
- Actor starts;
- unique free and paying users;
- run success rate;
- zero-result or failed user runs;
- which public task received runs;
- reviews, bookmarks, issues, or questions;
- top referrers and countries.

Do not create paid self-runs to inflate activity. Use only one tiny owner smoke run
when a deployment or source change needs verification.

Decision after seven days:

- keep and improve a task if it receives genuine starts or useful feedback;
- rewrite title/description if it receives views but no starts;
- replace the use case if it receives neither views nor starts;
- prioritize reliability fixes over more promotion if any user run fails.
